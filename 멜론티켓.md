# 좌석 배정 기능

## 구현 완료

### 배정 순서
- 예매 선착순 (뿌리오 데이터 `.reverse()` → 먼저 예매한 사람이 좋은 좌석)
- 결과 출력: VIP석 → R석 → S석 → A석 순서
- 결과 번호: 예매 순서 번호 (1번 = 가장 먼저 예매한 사람)

### 흐름
```
1. "최종결산" → 공연 목록 조회
2. "좌석배정1" → 해당 공연 선택 + 엑셀 업로드 대기
3. 미판매 좌석 엑셀 파일 텔레그램에 업로드
4. 봇이 파싱 → 좌석등급별 미판매 좌석 정리
5. 최종결산 데이터와 매칭 (등급 1:1 대응)
6. 예매 선착순 + 구역 우선순위 + 가운데 좌석 우선 + 같은 행 나란히 배정
7. 결과를 개인 봇으로 전송
```

### 구역 우선순위 (대구 콘서트하우스 그랜드홀)
```
1순위: B (정중앙)
2순위: A, C (중앙 좌/우)
3순위: E (중앙 하단)
4순위: I (중앙 상단)
5순위: J, H (상단 좌/우)
6순위: D, F (1층 사이드)
7순위: K, G (측면)
8~10순위: BL1~BL6 (발코니)
```

### 구역 우선순위 (울산문화예술회관 대공연장)
```
1순위: 1층 B구역 (정중앙)
2순위: 1층 A구역, 1층 C구역 (1층 좌/우)
3순위: 2층 B구역 (2층 중앙)
4순위: 2층 A구역, 2층 C구역 (2층 좌/우)
5순위: 3층 B구역 (3층 중앙)
6순위: 3층 A구역, 3층 C구역 (3층 좌/우)
```

### 엑셀 구조 (실측)
```
[0]No [1]이용일 [2]회차 [3]좌석등급 [4]층 [5]열(구역+행) [6]좌석수 [7]좌석번호
예: "BL5구역 1열" → section=BL5구역, row=1
```
