# 좌석 배정 기능

## 구현 완료

### 배정 순서
- 예매 선착순 (뿌리오 데이터 `.reverse()` → 먼저 예매한 사람이 좋은 좌석)
- 결과 출력: VIP석 → R석 → S석 → A석 순서
- 결과 번호: 예매 순서 번호 (1번 = 가장 먼저 예매한 사람)

### 흐름
```
1. "최종결산" → 공연 목록 조회
2. "좌석배정1" → 해당 공연 선택 + 엑셀 업로드 대기
3. 미판매 좌석 엑셀 파일 텔레그램에 업로드
4. 봇이 파싱 → 좌석등급별 미판매 좌석 정리
5. 최종결산 데이터와 매칭 (등급 1:1 대응)
6. 예매 선착순 + 구역 우선순위 + 가운데 좌석 우선 + 같은 행 나란히 배정
7. 결과를 개인 봇으로 전송
```

### 구역 우선순위 (대구 콘서트하우스 그랜드홀)
```
1순위: B (정중앙)
2순위: A, C (중앙 좌/우)
3순위: E (중앙 하단)
4순위: I (중앙 상단)
5순위: J, H (상단 좌/우)
6순위: D, F (1층 사이드)
7순위: K, G (측면)
8~10순위: BL1~BL6 (발코니)
```

### 구역 우선순위 (울산문화예술회관 대공연장)
```
1순위: 1층 B구역 (정중앙)
2순위: 1층 A구역, 1층 C구역 (1층 좌/우)
3순위: 2층 B구역 (2층 중앙)
4순위: 2층 A구역, 2층 C구역 (2층 좌/우)
5순위: 3층 B구역 (3층 중앙)
6순위: 3층 A구역, 3층 C구역 (3층 좌/우)
```

### 구역 우선순위 (창원 성산아트홀 대극장)
```
1순위: O열 (무대 바로 앞)
2순위: 1층 C열 (정중앙)
3순위: 1층 B열, 1층 D열 (중앙 좌/우)
4순위: 1층 A열, 1층 E열 (사이드)
5순위: 2층 C열 (2층 중앙)
6순위: 2층 B열, 2층 D열 (2층 좌/우)
7순위: 2층 A열, 2층 E열 (2층 사이드)
```

### 엑셀 구조 (실측)
```
[0]No [1]이용일 [2]회차 [3]좌석등급 [4]층 [5]열(구역+행) [6]좌석수 [7]좌석번호
예: "BL5구역 1열" → section=BL5구역, row=1
```

### 라벨 출력 순서
- VIP석 → R석 → S석 → A석 순서 (각 등급 내 선착순 유지)

### 세션 자동 재로그인
- 00:00 사전 알림 제거 (자동 재로그인이 처리)
- 쿠키 만료 시 ID/PW 자동 입력 → 2단계 인증 알림 → 핸드폰 승인 대기 (2분)
- ID/PW CONFIG에 직접 저장됨

### 문자 발송 방식 변경 (구현 완료)
**기존**: 뿌리오 "내 문자함"에서 지역명으로 템플릿 클릭 → 지역 같으면 구분 불가
**변경**: PERFORMANCES에 `sms` 정보(dateText, venue, addr) 추가 → 코드에서 직접 메시지 작성
- sms 정보 있으면: 직접 메시지 조합 (날짜/장소 정확)
- sms 정보 없으면: 기존 문자함 fallback (단, 같은 지역 2개+ 공연이면 발송 중단 + 텔레그램 알림)
- 현재 sms 등록: 대전_디즈니, 대전_지브리 (나머지는 추후 추가)

→ 재발송 건: 봇이 자동으로 다시 보내려면 6명의 **전화번호 + 좌석등급 + 수량**이 필요합니다.
뿌리오 발송결과에서 확인 가능하지만, 확실하게 하려면 직접 보내시는 게 안전할 수 있어요.
봇으로 보내고 싶으시면 6명의 전화번호 알려주세요!
