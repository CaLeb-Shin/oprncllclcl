# 좌석 배정 기능

## 구현 완료

### 배정 순서
- 예매 선착순 (뿌리오 데이터 `.reverse()` → 먼저 예매한 사람이 좋은 좌석)
- 결과 출력: VIP석 → R석 → S석 → A석 순서
- 결과 번호: 예매 순서 번호 (1번 = 가장 먼저 예매한 사람)

### 흐름
```
1. "최종결산" → 공연 목록 조회
2. "좌석배정1" → 해당 공연 선택 + 엑셀 업로드 대기
3. 미판매 좌석 엑셀 파일 텔레그램에 업로드
4. 봇이 파싱 → 좌석등급별 미판매 좌석 정리
5. 최종결산 데이터와 매칭 (등급 1:1 대응)
6. 예매 선착순 + 구역 우선순위 + 가운데 좌석 우선 + 같은 행 나란히 배정
7. 결과를 개인 봇으로 전송
```

### 구역 우선순위 (대구 콘서트하우스 그랜드홀)
```
1순위: B (정중앙)
2순위: A, C (중앙 좌/우)
3순위: E (중앙 하단)
4순위: I (중앙 상단)
5순위: J, H (상단 좌/우)
6순위: D, F (1층 사이드)
7순위: K, G (측면)
8~10순위: BL1~BL6 (발코니)
```

### 구역 우선순위 (울산문화예술회관 대공연장)
```
1순위: 1층 B구역 (정중앙)
2순위: 1층 A구역, 1층 C구역 (1층 좌/우)
3순위: 2층 B구역 (2층 중앙)
4순위: 2층 A구역, 2층 C구역 (2층 좌/우)
5순위: 3층 B구역 (3층 중앙)
6순위: 3층 A구역, 3층 C구역 (3층 좌/우)
```

### 구역 우선순위 (창원 성산아트홀 대극장)
```
1순위: O열 (무대 바로 앞)
2순위: 1층 C열 (정중앙)
3순위: 1층 B열, 1층 D열 (중앙 좌/우)
4순위: 1층 A열, 1층 E열 (사이드)
5순위: 2층 C열 (2층 중앙)
6순위: 2층 B열, 2층 D열 (2층 좌/우)
7순위: 2층 A열, 2층 E열 (2층 사이드)
```

### 엑셀 구조 (실측)
```
[0]No [1]이용일 [2]회차 [3]좌석등급 [4]층 [5]열(구역+행) [6]좌석수 [7]좌석번호
예: "BL5구역 1열" → section=BL5구역, row=1
```

### 라벨 출력 순서
- VIP석 → R석 → S석 → A석 순서 (각 등급 내 선착순 유지)

### 문자 발송 방식
- **기존 방식 유지**: 뿌리오 "내 문자함"에서 지역명 템플릿 클릭 → 변수 교체 → 발송
- **추가**: 같은 지역 공연 2개 이상이면 텔레그램 알림 (발송 전 확인 요청)

### 문자함 템플릿 다중 페이지 대응 (해결)
- 템플릿 7개 이상이면 2페이지로 넘어감
- 3단계 시도: 1페이지 직접 찾기 → 검색 → 페이지 번호 클릭

### 같은 지역 공연 알림 개선 (해결)
- 기존: PERFORMANCES config 기반 → 문자함에서 삭제해도 매번 알림
- 변경: 문자함 페이지에서 실제 지역 템플릿 개수를 세고, 2개 이상일 때만 알림

### 세션 만료 시 대응
- 세션 만료 시 텔레그램 알림 → 서버에서 `node setup-login.js` 수동 실행
- 뿌리오는 네이버 OAuth 쿠키로 자동 재로그인 시도 (실패 시 알림)

### 좌석배정 정렬 개선 (완료)
- 층이 다르면 낮은 층 무조건 우선 (1층 > 2층)
- 같은 층, 3열 이상 차이: 앞열 우선 (A구역 1열 > B구역 4열)
- 같은 층, 2열 이하 차이: 구역 우선순위 (B구역 3열 > A구역 1열)
- 같은 구역이면 항상 앞열 우선
- 결과에 총 배정 인원 + 좌석수 표시
