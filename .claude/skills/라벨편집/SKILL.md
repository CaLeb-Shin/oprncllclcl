# /라벨편집 — 명단 → 라벨 표준 형식 정리

사용자가 올린 **좌석 스크린샷** 또는 **자유 텍스트 명단**을, 텔레그램 봇의
`/라벨추가` 명령에 그대로 붙여넣을 수 있는 **표준 형식**으로 정리한다.

역할 분담: 지저분한 입력 해석은 여기(Claude)가 하고, PDF 출력은 봇이 한다.
이 스킬의 결과물(표준 형식 텍스트)을 사용자가 복사 → 텔레그램 봇에서
`/라벨추가` 입력 후 붙여넣으면 봇이 라벨 PDF(글로리텍 8189)를 만든다.

## 입력 (세 가지 형태)
1. **좌석배치도 스샷** — 이름이 `홍길동(3명)` 식으로 박스에 적혀 있음
2. **주문관리 표 스샷** (인터파크/티켓 주문 목록) — 행마다 이름·핸드폰·상품명·티켓번호들
3. **자유 텍스트 명단** — 카톡/메모에서 복사
- 함께 오는 **좌석배정 지시** (예: "담당자2명은 vip로, 나머지는 R석으로")
- "쿠팡꺼" 같은 **채널 표시** (쿠팡이면 라벨을 기울임으로 구분)

## 출력 — 표준 형식 (봇 /라벨추가 입력과 동일해야 함)
한 줄 = 1명. 코드블록으로 감싸서 복사하기 쉽게 출력한다.
```
이름[(뒷4자리)] [좌석등급] N매
```
예시:
```
담당자 VIP석 2매
박근해 R석 3매
김송희(3586) VIP석 3매
이창준(4594) 4매
뀨 5매
```
- 뒷4자리: 있으면 `이름(1234)`, 없으면 괄호 없이 `이름`
- 좌석등급: `VIP석` / `R석` / `S석` / `A석`. **없으면 그냥 생략** (`이름 N매`)
- 매수: `N매`. 인원만 있으면 그게 매수 (예: `홍길동(3명)` → `3매`)

### 쿠팡 등 채널 구분 → `[쿠팡]` 그룹 헤더
쿠팡 주문은 라벨을 **기울임**으로 구분한다. 표준 형식에서는 `[쿠팡]` 헤더
아래에 묶으면 봇이 그 줄들을 자동으로 기울여 출력한다. 다른 채널/일반은
헤더 없이(또는 `[펄스]` 등 다른 헤더로) 둔다.
```
[쿠팡]
전율리아(1177) VIP석 5매
강보미(4405) VIP석 6매

일반손님(5678) R석 2매
```
- 헤더에 "쿠팡"(또는 coupang)이 들어가야 기울임 적용됨. 다른 헤더는 기울임 안 됨.

## 추출·정리 규칙 (사용자 확정)
1. **홀드 제외**: "홀드"/"hold"/"보류" 표시된 항목은 라벨에서 **뺀다**.
2. **좌석등급 없으면 없는 대로**: 등급 정보가 없으면 채우지 말고 생략.
3. **좌석배정 지시 적용**: "담당자는 VIP, 나머지는 R석" 같은 지시를 각 사람에
   매핑. 지시에 없는 사람은 등급 미지정(생략).
4. **인원 합산**: `장애인1분 일반인2분` 같은 표기는 합쳐서 매수로 (→ 3매).
5. **담당자:실구매자 형태**: `또희 : 김송희` 처럼 "담당자 : 실구매자"면
   **실구매자 이름**을 쓴다.
6. **배정 메모 제외**: "네이버 2층 통로쪽 제일 앞 배정" 같은 좌석배정 지시
   메모는 라벨에 넣지 않는다 (이름/등급/매수만).
7. **그룹 헤더 무시**: `[쿠팡]`, `[ 펄스 ]` 같은 채널 구분 줄은 라벨 대상 아님.
8. 이미지 인원수 `(N명)`은 매수 N으로 해석.

## 주문관리 표 스샷 (인터파크/티켓 주문 목록) — 별도 규칙
네이버 스토어 표가 아니라 **티켓 주문관리 표**를 올린 경우:
- **뒷4자리 = 핸드폰 번호 끝 4자리** (예: `010-8885-3202` → `3202`)
- **매수 = 그 사람의 티켓 행 개수** (구매자 한 명 아래 티켓번호 줄을 센다)
- **좌석등급 = 상품명의 `/ ___석` 뒤 선택 옵션**
  - 예: `... (비지정석) VIP석,R석 / VIP석 16:00` → **VIP석**
  - 예: `... (비지정석) R석 / R석 16:00` → **R석**
- 취소/반품 상태 행은 제외, "예약확정"만 집계
- 쿠팡 주문이면 결과를 `[쿠팡]` 그룹으로 묶어 기울임

예) 위 표에서 `최성오 010-8885-3202` + VIP석 티켓 2장 → `최성오(3202) VIP석 2매`

## 정렬
봇이 등급순(VIP→R→S→A)으로 자동 정렬하므로, 출력 순서는 크게 상관없다.
읽기 좋게 등급순 또는 입력순으로 둔다.

## 결과 보고 형식
1. **✅ 붙여넣기용** — 표준 형식 코드블록 (이게 봇에 붙여넣을 본체).
   채널별로 `[쿠팡]` / 일반 그룹으로 묶어서 출력.
   **그룹 헤더 옆에 그 그룹의 총 매수를 적는다** — 예: `[쿠팡 · 20매]`,
   `[파주교육지원청 · 33매]`. (봇은 `[...]` 헤더 내용을 무시하고 "쿠팡" 포함
   여부만 보므로 매수를 적어도 파싱에 지장 없음.)
2. **요약** — 채널 그룹별로 **인원·총 매수**를 적고, 전체 합계도 적는다. 예:
   - 🛒 쿠팡: 6명 / 20매
   - 일반: 1명 / 1매
   - **합계: 7명 / 21매**
   (쿠팡 그룹의 총 매수를 꼭 표시. 단, 라벨지에는 "쿠팡" 글자를 넣지 않고
   기울임으로만 구분되므로 그 점도 한 줄 안내한다.)
3. **⚠️ 확인 필요** — 애매했던 항목(홀드라 뺀 것, 등급 못 정한 것, 담당자/실구매자
   판단 등)을 표로 정리해 사용자에게 확인 요청. 임의로 버리지 말고 반드시 알린다.

## 사용 흐름 안내 (결과 끝에 덧붙이기)
> 위 "붙여넣기용" 블록을 복사 → 텔레그램 봇에 `/라벨추가` 입력 →
> 봇이 "명단 주세요" 하면 붙여넣기 → 라벨 PDF 수신.

## 주의
- 이름·뒷4자리는 **추측하지 말 것**. 이미지가 흐려 불확실하면 ⚠️에 표시하고 묻는다.
- 좌석등급을 임의로 채우지 말 것 (지시 없으면 생략).
